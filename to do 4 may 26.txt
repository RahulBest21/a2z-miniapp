═══════════════════════════════════════════════════════════════

This phase implements the question tagging taxonomy (Subject → Chapter
→ Topic → Subtopic) and the per-user analytics aggregation that drives
the "AI Analysis" screens. All computation is deterministic (SQL + Python);
Gemini is only used for initial question tagging during rephrase.

[ ] TAX-01 | a2z_db.py | Taxonomy Tables Schema
      └─ Create new tables:
         [ ] 1a. subjects (id, name, exam_category)
                └─ e.g. Quantitative Aptitude, Reasoning, English, GA, Vocab
         [ ] 1b. chapters (id, subject_id FK, name, order_index)
                └─ e.g. Arithmetic, Algebra, Geometry under Quant
         [ ] 1c. topics (id, chapter_id FK, name, weightage, order_index)
                └─ e.g. Percentage, Ratio, Time & Work under Arithmetic
         [ ] 1d. subtopics (id, topic_id FK, name, order_index)
                └─ e.g. "LCM-based", "Ratio-based" under Time & Work
      └─ Seed default taxonomy for SSC CGL, UPSC, Banking exams.

[ ] TAX-02 | a2z_db.py | Add Tag Columns to Questions Table
      └─ ALTER TABLE questions ADD COLUMN:
         [ ] 2a. subject_id INTEGER FK → subjects
         [ ] 2b. chapter_id INTEGER FK → chapters
         [ ] 2c. topic_id INTEGER FK → topics
         [ ] 2d. subtopic_id INTEGER FK → subtopics
         [ ] 2e. difficulty_level TEXT (easy/medium/hard)
         [ ] 2f. cognitive_level TEXT (knowledge/application/analysis)
         [ ] 2g. concept_tags TEXT (JSON array of extra tags)
      └─ Backfill existing questions with default tags.

[ ] TAX-03 | mainmock.py | Gemini Tagging During Rephrase
      └─ Update SYSTEM_PROMPT in Gemini rephrase pipeline:
         [ ] 3a. Add instruction: "For each question, also return:
                subject, chapter, topic, subtopic, difficulty_level,
                cognitive_level, and concept_tags."
         [ ] 3b. Parse tags from Gemini response alongside rephrased text.
         [ ] 3c. Normalize tags to existing taxonomy (or create new entries).
         [ ] 3d. Store tags on questions table during register_questions().
      └─ Gemini model: gemini-3.1-flash-lite-preview (as per Rahul+Ajay decision).

[ ] ANA-01 | a2z_db.py | User Analytics Aggregation Tables
      └─ Create aggregation tables (updated on every mock submit):
         [ ] 1a. user_topic_stats
                (user_id, topic_id, subject_id, chapter_id,
                 attempts, correct, wrong, skipped,
                 accuracy REAL, last_attempted_at TEXT)
         [ ] 1b. user_subject_stats
                (user_id, subject_id,
                 attempts, correct, wrong, skipped,
                 accuracy REAL, last_attempted_at TEXT)
         [ ] 1c. user_chapter_stats
                (user_id, chapter_id, subject_id,
                 attempts, correct, wrong, skipped,
                 accuracy REAL, last_attempted_at TEXT)
         [ ] 1d. user_mock_group_stats
                (user_id, group_id, mocks_attempted,
                 avg_score REAL, best_score REAL, latest_score REAL,
                 trend_json TEXT, last_attempted_at TEXT)
      └─ All stats are deterministic counts, updated via trigger/function.

[ ] ANA-02 | a2z_db.py | Analytics Aggregation Functions
      └─ [ ] 2a. update_user_topic_stats(user_id, mock_id)
             └─ Called after submit_attempt(). Aggregates responses
                by topic/subtopic and updates all stats tables.
      └─ [ ] 2b. get_user_topic_breakdown(user_id, subject_id=None)
             └─ Returns topic-wise: attempts, correct, wrong, skipped, accuracy.
      └─ [ ] 2c. get_user_subject_summary(user_id)
             └─ Returns subject-wise: total attempts, accuracy, strong/weak.
      └─ [ ] 2d. get_user_score_trend(user_id, group_id=None, limit=10)
             └─ Returns score history: mock_title, score, max, accuracy, date.
      └─ [ ] 2e. get_user_weak_areas(user_id, min_attempts=3, top_n=5)
             └─ Returns topics with lowest accuracy where attempts ≥ min_attempts.
      └─ [ ] 2f. get_user_silly_mistakes(user_id, mock_id=None)
             └─ Detects easy questions answered wrong (easy=1, correct=0).

[ ] ANA-03 | mainmock.py | Analytics Endpoints for Mini App
      └─ Add _webapp_data actions for analytics:
         [ ] 3a. get_my_analytics → returns full analytics JSON
                (subject_summary, topic_breakdown, weak_areas,
                 score_trend, silly_mistakes, strongest_areas)
         [ ] 3b. get_mock_analysis → returns per-mock analysis JSON
                (score, accuracy, time_breakdown, topic_performance,
                 question_list with tags)
         [ ] 3c. get_group_analysis → returns series-level analytics
                (mocks_attempted, avg_score, best_score, trend,
                 subject_comparison)

[ ] ANA-04 | mainmock.py | Per-Question Response Tracking
      └─ Wire existing responses table into actual mock submission flow:
         [ ] 4a. In submit_attempt: iterate all questions, record each
                response with: selected_option, is_correct, time_spent_sec.
         [ ] 4b. Populate question tags (subject/chapter/topic/subtopic)
                from question row into response for fast aggregation.
      └─ This enables: per-question review, topic-wise stats, time analysis.

PHASE 5B — AI INSIGHTS LAYER (Deterministic + Gemini Polish)  (P2-P3)
═══════════════════════════════════════════════════════════════

"AI Analysis" that LOOKS like premium AI but is actually deterministic
data analysis with optional Gemini-generated Hinglish copy.

[ ] AI-01 | mainmock.py | Template-Based Insights Engine (FREE)
      └─ Deterministic analytics → human-readable text:
         [ ] 1a. classify_accuracy(acc) → "Strong" (>70%), "Moderate" (40-70%), "Weak" (<40%)
         [ ] 1b. For each subject/topic, map to pre-written Hinglish templates:
              └─ Strong: "{topic} me aap strong hain (accuracy {acc}%).
                 Weekly revision se maintain karo."
              └─ Weak+HighAttempts: "{topic} me bahut attempts hain
                 lekin accuracy sirf {acc}%. Conceptual clarity improve karo."
              └─ Weak+LowAttempts: "{topic} pe focus badhao. Abhi tak
                 sirf {attempts} sawaal kiye hain."
         [ ] 1c. combine_insights(analytics_json) → 4-5 line paragraph
         [ ] 1d. Time management tips from time_spent data
              └─ "Jab aap 30 sec se kam lete ho, accuracy 38% gir jaati hai."
         [ ] 1e. Stamina indicators: first-half vs second-half performance

[ ] AI-02 | mainmock.py | Gemini-Powered Insights (PREMIUM)
      └─ When Gemini quota available, use LLM for natural Hinglish:
         [ ] 2a. Prompt template:
              "You are an exam performance coach for SSC/RRB students.
               Based on this JSON data, write 3-4 lines personalized
               feedback in Hinglish. Focus on weakest topics first,
               then time management tips. Be encouraging but direct.
               JSON: {analytics_json}"
         [ ] 2b. Cache insights for 24h to avoid re-generation
         [ ] 2c. Fallback to template-based if Gemini fails/quota exhausted
      └─ Gemini model: gemini-3.1-flash-lite-preview

[ ] AI-03 | mainmock.py | Mock-Level AI Analysis Generator
      └─ After mock submit, generate "AI Mock Analysis":
         [ ] 3a. Aggregate per-question responses
         [ ] 3b. Compute: silly mistakes, guessing patterns, speed issues
         [ ] 3c. Generate 2-line Hinglish summary per subject
         [ ] 3d. Return as JSON → stored in attempts table (ai_analysis_json)
         [ ] 3e. Mini app renders in "AI Analysis" tab

[ ] AI-04 | mainmock.py | AI Insights Endpoints
      └─ _webapp_data actions:
         [ ] 4a. get_ai_insights → returns generated insights JSON
                (mock_level, subject_level, overall_level)
         [ ] 4b. refresh_ai_insights → forces regeneration (bypass cache)
         [ ] 4c. get_study_plan → AI-generated weekly plan based on analytics
      └─ All insights stored with timestamp + cache_key for reuse

[ ] GAME-01 | mainmock.py | Raid Boss System
      └─ "🚨 RAID EVENT: Boss HP: 100. Every new user quiz solve = -1 HP"
      └─ a2z_gamification.py already has create_raid + deal_raid_damage.
         Wire into bot:

         [ ] 1c. Raid status broadcast to channel
